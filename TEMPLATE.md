复制本文件到 `episodes/`，重命名为 `{三位序号}-{短标题}.md` 后填写。Markdown 是单一可信源；需要导演预览页时，从 `docs/storyboard-template.html` 复制到 `production/{三位序号}-{短标题}/storyboard.html`。

参考：[可行性方案](docs/可行性方案.md)、[制作指南](docs/制作指南.md)、[角色库](docs/characters.md)、[场景库](docs/locations.md)。

## 6. 分镜语言单表

命名采用 **x场x镜**，素材文件与场镜号一致。`i2v?` 只标必须图生视频的镜头；能用截图、字幕、推拉、定格完成的镜头标 No。

字段约定：

- `role`：`setup` / `tension` / `escalation` / `pause` / `punchline` / `aftermath`。
- `情绪强度 1-5`：默认 2；越高越接近爆发或强 punchline。
- `运动强度`：`none` / `low` / `mid` / `high`，默认 `low`。
- `cue_id`：引用本期 `production/{序号}/audio/cue-sheet.md` 中的声音 ID。

| 项目 | 内容 |
|------|------|
| **画幅** | 9:16 / 16:9 / 条漫 |
| **风格参考** | `--sref` / 参考图 / 主角种子图 |
| **色调** | （各场主色与对比关系） |
| **总时长** | （秒） |
| **i2v 镜数** | N / 总镜数 |

## 9. 高阶：可选 `[meta]` 块

简单镜头只填上面的分镜单表即可；复杂镜头、需要自动化处理的镜头，可以额外补一个 `[meta]` 块。不要强制所有镜头都写，也不要把它做成 yaml front matter。

```yaml
[meta]
shot: close-up
angle: low
camera_motion: dolly_zoom
duration: 5s
role: tension
emotion: { facial: annoyed, intensity: 4 }
motion_intensity: high
character: [employee_01]
location: office_floor_5
audio: [tape_rip, office_ac_hum]
```

使用规则：

- 简单镜头不写 `[meta]`。
- 只有复杂动作、角色一致性要求高、或未来要交给自动化脚本时才写。
- `[meta]` 的值应与分镜单表保持一致，以分镜单表为准。
